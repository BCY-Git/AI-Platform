一、视觉设计
（一）色彩体系
浅色模式
主色调：采用柔和的蓝绿色调，营造清新、友好且专业的氛围，使其区别于豆包现有的蓝色调，在竞品中突出自身特色。
辅助色：搭配浅粉色作为强调色，为界面增添温暖与活力，用于突出重要信息、交互元素，如按钮悬停、重要提示等场景。
中性色：白色作为背景色，呈现简洁干净的视觉感受；浅灰色用于分隔线和次要文本，降低视觉干扰；黑色用于重要文本，确保信息清晰可读，通过色彩层次构建出明确的视觉层级。
深色模式
主色调：将蓝绿色调调整为较深的同色系，在深色背景下保持独特辨识度，传递沉稳、专业的气质，满足部分用户在暗光环境下的使用需求或对深色风格的偏好。
辅助色：适当加深浅粉色，使其在深色环境中依然能够醒目突出，继续承担强调重要交互元素的功能，如在深色模式下的重要按钮、提示信息等场景。
中性色：背景采用深灰色或黑色，营造深邃的视觉效果；浅灰色用于文本和分隔线，保证信息的可读性；白色用于强调关键信息，增强视觉对比，使重要内容更加突出。
（二）字体选择
选用现代、简洁且易读的无衬线字体，如 Roboto 或 Inter。
浅色模式：标题和重要信息使用深灰色或黑色粗体字，增强视觉冲击力，吸引用户注意力；正文部分采用适中的字重，配合合理设置的字体大小、行间距和字间距，确保长时间阅读的舒适性，避免视觉疲劳。
深色模式：标题和重要信息运用白色或浅灰色粗体字，在深色背景下脱颖而出，强化信息层级；正文采用浅灰色，与深色背景形成良好对比，同时调整行间距和字间距，以适应深色视觉环境下的阅读需求，保证阅读体验的流畅性。
（三）图标设计
设计一套风格统一的自定义图标，整体采用简洁、扁平的设计风格，与整体界面风格高度契合。
浅色模式：图标线条简洁流畅，颜色与整体风格协调匹配，部分交互图标在用户悬停时，会呈现与主色调呼应的动效，如颜色变化、轻微旋转或淡入效果，增加趣味性和交互性。
深色模式：图标线条颜色变浅，部分图标填充色与深色背景形成鲜明对比，提高识别度。交互图标动效颜色也相应调整，选择与深色模式协调的色彩，如金色或亮蓝色，使动效在深色背景下更加醒目，增强视觉效果和用户操作反馈。
二、布局与交互
（一）首页布局
搜索栏：浅色与深色模式下，均将搜索栏置于页面顶部显著位置，采用较大尺寸和醒目的样式，使其成为用户进入页面后的首要视觉焦点。搜索栏旁边设置语音输入按钮，按钮上配备动态的麦克风图标，实时提示用户语音输入功能的可用性。在深色模式下，搜索栏背景和边框颜色与整体深色风格适配，输入提示文字颜色调整为浅灰色，确保清晰可读。
推荐内容展示：在搜索栏下方，以卡片形式展示热门话题、热门问题及精彩回答。每张卡片尺寸较大，突出标题和关键信息，并配以相关小图标或缩略图，增强视觉吸引力。卡片采用轻微阴影效果，营造层次感，在深色模式下阴影效果适当减弱，避免视觉过重。用户可左右滑动浏览不同类别的推荐卡片，点击卡片可直接查看详细内容。
快捷功能区：页面底部设置一排快捷功能按钮，包括 “提问”“我的收藏”“历史记录” 等常用功能。按钮尺寸较大，图标清晰，易于点击操作。在深色模式下，按钮图标颜色与背景色对比度增强，确保在深色环境中按钮的可识别性和可点击性。用户点击按钮时，按钮会产生短暂的放大或变色动画，给予明确的操作反馈。
（二）提问页面
输入框：输入框占据页面大部分空间，浅色模式下采用白色背景和浅灰色边框，输入框内有清晰的提示文字引导用户输入问题。当用户开始输入时，提示文字自动消失，输入框边框颜色变为与主色调一致的颜色，实现视觉聚焦。深色模式下，输入框背景色变为深灰色，边框颜色变为浅灰色，输入提示文字颜色调整为浅灰色，当用户输入时，边框颜色同样变为与主色调深色版本一致的颜色。
辅助功能：在输入框下方，提供 “添加话题标签”“选择问题分类” 等辅助功能，以小按钮或下拉菜单形式呈现。在深色模式下，辅助功能的背景和文字颜色进行相应调整，确保操作区域与深色背景区分明显，方便用户准确描述问题，提高问题曝光度和回答质量。
提问按钮：“提问” 按钮置于页面右下角，浅色模式下采用与主色调一致的醒目的颜色，按钮上搭配绿色对勾图标，暗示提问操作的确认。当用户点击按钮时，按钮会出现加载动画，直至问题成功提交。深色模式下，按钮颜色调整为与主色调深色版本一致，对勾图标颜色可变为金色或白色，突出按钮的可点击性，加载动画效果也与深色风格相适配。
（三）回答页面
回答内容展示：回答内容以清晰的层级结构呈现。浅色模式下，答案文本采用较大字号和适中行间距，易于阅读；回答者信息（头像、用户名、认证信息等）置于答案顶部，采用较小字号和浅灰色字体，与答案内容区分开来。对于优质回答，添加特殊样式突出显示，如在回答卡片周围添加金色边框，或在回答者信息旁添加 “优质回答” 标签。深色模式下，回答卡片背景色调整为深灰色，优质回答的金色边框更加醒目；答案文本颜色变为浅灰色或白色，与背景形成鲜明对比；回答者信息颜色变为浅灰色，确保信息层级清晰。
互动功能：每个回答下方设置点赞、评论、分享等互动按钮。浅色模式下，按钮采用简洁图标形式，用户操作时，按钮会产生相应动画反馈，如点赞按钮点击后变成实心红色爱心，同时数字实时更新。深色模式下，按钮图标颜色和点击动效颜色进行调整，使其在深色背景下视觉效果更佳，例如点赞按钮点击后红色爱心颜色更加鲜艳，与深色背景形成强烈对比，增强操作反馈效果。评论区域采用展开 / 收起形式，默认收起以保持页面简洁，用户点击 “查看评论” 按钮后可展开查看全部评论，并进行回复操作。在深色模式下，评论区域的背景和文字颜色适配深色环境，展开 / 收起按钮的颜色和图标也相应调整，确保清晰可见。
相关问题推荐：在回答页面底部，展示与当前问题相关的其他问题及回答，引导用户进一步探索相关知识。相关问题以列表形式呈现，每个问题仅显示标题和简短预览内容，用户点击可查看完整问题和回答。在深色模式下，相关问题列表的背景和文字颜色与深色风格统一，确保在深色环境下的可读性和视觉协调性。
（四）模式切换入口
在页面右上角设置专门的模式切换按钮，按钮图标设计为太阳和月亮的组合（白天显示太阳图标，夜晚显示月亮图标），直观传达切换功能。用户点击按钮时，按钮会产生短暂旋转动画，随后页面以渐变动画平滑过渡到另一种模式，所有元素的颜色、背景色和图标颜色等逐渐变化，避免突然切换给用户带来不适感。
三、动效与微交互
（一）页面切换动效
浅色模式下：当用户在不同页面之间切换时，采用流畅的淡入淡出或滑动动画。例如，从首页进入提问页面时，提问页面从底部向上滑动进入视野，同时首页向下滑动消失；返回首页时，则反之，使页面过渡自然，减少突兀感。
深色模式下：同样采用流畅的页面切换动效，但动效的颜色和视觉效果与深色背景相协调。例如，页面切换时的淡入淡出效果，其过渡颜色与深色环境相融合，避免出现过于刺眼或不协调的视觉变化。
（二）元素加载动效
浅色模式：对于页面上的元素，如卡片、图标等，采用逐帧加载或淡入的动效。比如，推荐卡片加载时从左到右依次淡入显示，给用户一种内容逐步呈现的感觉，让用户在等待内容加载过程中感受到流畅的视觉体验。
深色模式：元素加载动效根据深色背景进行调整。例如卡片加载可采用从中心向外扩散的淡入效果，使动效在深色背景下更具视觉吸引力，同时避免与深色背景产生视觉冲突，确保加载过程的视觉流畅性和舒适性。
（三）微交互
浅色模式：在细节处添加微交互提升用户体验。如用户将鼠标悬停在按钮上，按钮会有轻微放大和变色效果，提示按钮可点击；用户成功提交问题后，页面出现短暂庆祝动画，如 confetti 飘落或星星闪烁，给予积极反馈。
深色模式：微交互效果在深色背景下进行优化。按钮悬停和点击动效的颜色选择与深色模式协调的色彩，确保动效在深色背景下清晰可见且不刺眼。成功提交问题后的庆祝动画颜色和效果也进行相应调整，使其与深色环境相融合，同时保持足够的视觉吸引力，强化用户的积极体验。
